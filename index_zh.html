会议征集

研讨会的目标是什么？

研讨会的基本目标是召集机器学习工具和框架提供者及 Web 平台从业者共同探讨如何为机器学习在开放Web平台上的应用提供更好的基础支撑。

研讨会的其它目标如下：

  • 理解机器学习如何适应 Web 技术栈
  • 理解基于浏览器的机器学习如何适应机器学习生态系统
  • 探索机器学习技术对 Web 浏览器和 Web 应用的影响
  • 评估制定机器学习 API 和标准化范式的可能性

参会者在听取报告的同时，可以积极参与到分组话题讨论中，一起探讨相关的与会话题。

研讨会的主题有哪些？

  • 是否需要基于浏览器的专用机器学习 API ？这样的浏览器级的 API 需要什么样的水平？在市场上部署机器学习专用处理器（如 NPU,TPU）有怎样的前景？
  • 怎样将机器学习的 API 和框架与数据源结合（包括传感器 API,从设备和 WebRTC 流中输入的音频和视频）？
  • 如何能将机器学习置于已有的专用高性能计算 API 之上（如 WebGL, WebGPU, WebAssembly 等）
  • 怎样将机器学习原有方法关联到特定领域的人工智能 API 上（如声音识别，形状识别）？
  • 训练好的机器学习模型存在哪些交换格式？为将它们应用于 Web 浏览器，还需要一些什么样的特性？
  • 基于云端和客户端的机器学习可以如何交互以适应用户环境（如网络、可用计算力、可用电池电量）并同时保护用户隐私？单一的机器学习 API 是否无法同时满足云端和客户端需求？
  • 机器学习模型需要完成多少训练(如果需要训练)以有效地应用于浏览器中？
  • 从隐私和安全角度考虑，机器学习在浏览器中应用会有哪些优势和风险？怎样减轻这些风险？
  • 如何将机器学习原有方法应用于浏览器以帮助提升 Web 页面和 Web 应用的无障碍性？

对研讨会的主题有任何意见或建议，请通过向 Github 提交请求或向 Dominique Hazael-Massieux < dom@w3.org> 发送电子邮件。

如何参会？

研讨会对公众开放，不限于 W3C 会员。所有受邀人员均可免费参会。

如果您有兴趣参与研讨会，请填写该注册表。 我们欢迎对机器学习和 Web 技术的交叉应用有深刻见解的人员，不论是有解决该类问题的实践经验，或是参与过相关的标准化制定。

由于会场空间有限，最多可容纳120位参会人员，因此仅接受收到参会许可邮件的人员参会。 另外，请密切关注重要日期。

  • 机器学习硬件和芯片提供商

如何提交会议报告？

本次活动是一次研讨会，因此任何报告演示都应该尽可能地在短时间内完成，切合所提议的且由会议主席和程序委员会确定的主题。

为了能更好地在研讨会期间开展高效讨论，我们鼓励所有参会人员在参会前提前阅读所有被接收的主题。

如果您有意针对某一主题进行报告展示，请在截止日期(请查看重要日期)前通过邮件 <group-machine-learning-pc@w3.org> 向程序委员会提交您的报告。 我们的程序委员会会对您提交的材料进行评审并为您选择最相关的主题和方向。

报告准备应包含若干段落，且涵盖以下信息：

  • 您在网络和 Web 技术方面的背景
  • 您想引导大家对哪个主题进行讨论
  • 相关支撑材料或资源的链接
  • 为保证研讨会的效果，您认为研讨会还应该包含的其他主题
  • 提交的报告必须是英文的，最好是 HTML 或纯文本格式。 如果含有图片，请保证图片在 HTML 中是使用 base64 编码的 URI 内联的。 您可以涉及多个主题，但请将所有主题涵盖到同一份报告准备中，对于每位参会者我们仅接受一份报告。 所有在注册时提交的材料（如引用，目标，兴趣点等）都将进行发布并在本页面提供链接

什么是 W3C？

W3C 是一个世界范围内开放的标准组织，汇集众多公司与社区成员共同围绕现有和新兴技术展开富有成效的讨论，其为 Web 标准设立了免版税的专利框架。W3C 主要关注客户端（浏览器）技术，并且拥有成熟的词汇表（或“本体”）开发历史。W3C 根据成员和社区输出的优先级开展相应标准化工作。
